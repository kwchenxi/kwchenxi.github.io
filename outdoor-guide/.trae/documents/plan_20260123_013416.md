# 将户外向导应用转换为 InstantDB 全栈应用

## 阶段 1：设置和配置
1. **安装 InstantDB SDK**
   - 安装 `@instantdb/react` 包
   - 配置环境变量（使用 App ID: 74a1e87c-3632-43a5-9a6b-24a9d8d2532a）

2. **创建数据库配置**
   - 创建 `src/db/config.ts` 文件
   - 初始化 InstantDB 客户端
   - 定义 schema（7个表：trails, routeSegments, routeNodes, gear, users, favorites, corrections）

## 阶段 2：数据层迁移
3. **创建数据库服务层**
   - 创建 `src/db/trailService.ts` - 路线 CRUD 操作
   - 创建 `src/db/userService.ts` - 用户和收藏管理
   - 创建 `src/db/correctionService.ts` - 纠错记录管理
   - 实现与 InstantDB 的交互（查询、事务）

4. **数据迁移工具**
   - 创建脚本将 localStorage 数据迁移到 InstantDB
   - 保留现有数据完整性

## 阶段 3：UI 功能集成
5. **发布路线功能**
   - 在 `TrailGuideView.tsx` 中添加"发布到社区"按钮
   - 创建路线发布表单组件
   - 实现路线上传到数据库

6. **收藏路线功能**
   - 在路线详情页添加收藏按钮
   - 创建收藏列表页面
   - 实现实时收藏状态同步

7. **路线纠正功能**
   - 扩展现有纠错模式
   - 记录纠错历史到数据库
   - 添加纠错审核机制

8. **社区路线浏览**
   - 创建社区路线列表页面
   - 添加搜索和筛选功能
   - 显示路线贡献者信息

## 阶段 4：优化和测试
9. **实时同步**
   - 利用 InstantDB 实时功能
   - 实现多端数据同步
   - 添加离线支持

10. **测试和部署**
    - 测试所有 CRUD 操作
    - 验证数据一致性
    - 更新部署配置

## 技术栈
- **数据库**: InstantDB (App ID: 74a1e87c-3632-43a5-9a6b-24a9d8d2532a)
- **前端**: React + TypeScript + Vite (现有)
- **AI**: Gemini 2.0 Flash (现有)
- **状态管理**: InstantDB React Hooks

✅ 用户可以纠正和改进路线信息

✅ 实时数据同步
